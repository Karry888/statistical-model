# 数据源重构：消费端如何驱动上游清洁化

## 1. 研究主线与识别逻辑

研究主线：
消费端低碳偏好（政策约束、品牌采购、终端市场溢价）
→ 中游采购结构变化（低碳铝、绿电铝占比上升）
→ 上游生产响应（产能迁移、绿电替代、工艺优化）
→ 单吨排放与总排放变化。

围绕这条链路，数据不再只聚焦“产能是否转移”，而是要同时覆盖：
1. 消费端压力与偏好
2. 传导机制（采购与价格）
3. 上游响应（生产与能源）
4. 减排结果（碳强度与总量）

---

## 2. 数据模块与采集清单

### A. 消费端需求与约束（核心驱动变量）

核心问题：消费侧“为什么会要低碳铝”？

必收数据：
1. 终端行业低碳约束强度
	- 指标：汽车、光伏、家电、包装等行业的碳核算要求、绿色采购要求、披露频率
	- 频率：年度/季度
	- 来源：行业协会、龙头企业ESG报告、供应链采购标准
2. 出口导向需求（CBAM/海外客户）
	- 指标：对欧出口规模、CBAM覆盖产品占比、隐含碳申报要求
	- 频率：月度/季度
	- 来源：海关、欧盟CBAM规则文本、企业公告
3. 消费端低碳支付意愿或溢价
	- 指标：低碳铝与常规铝价差、绿色订单占比
	- 频率：周度/月度
	- 来源：SMM、Mysteel、企业采购公告

你当前状态：
- 待重点补齐（目前目录中无系统化消费端数据）。

### B. 中游传导机制（把“需求”连到“上游”）

核心问题：消费端压力如何传到电解铝企业？

必收数据：
1. 铝加工与制造端采购结构
	- 指标：低碳铝采购占比、长协比例、供应商地区结构
	- 频率：年度/半年度
	- 来源：上市公司年报、可持续发展报告、招标文件
2. 价格传导
	- 指标：电解铝价格、加工费、低碳铝溢价、绿电交易价格
	- 频率：日度/月度
	- 来源：SMM、上期所、各地电力交易中心
3. 订单约束强度
	- 指标：客户对碳足迹认证要求（如ASI、ISO 14067）、供货准入门槛
	- 频率：事件/年度
	- 来源：企业公告、认证机构数据库

你当前状态：
- 有电价基础，但缺“采购结构”和“低碳订单约束”数据。

### C. 上游响应（生产布局与能源替代）

核心问题：上游是否因消费侧压力发生实质性清洁化？

必收数据：
1. 产能与产量响应
	- 指标：分省运行产能、建成产能、产量、开工率
	- 频率：月度/季度/年度
	- 来源：SMM、统计年鉴、企业公告
2. 能源结构响应
	- 指标：分省电网排放因子、绿电占比、自备电占比、市场化购电结构
	- 频率：年度（排放因子）+月度（电量结构，若可得）
	- 来源：生态环境部、国家发改委、各省能源局、电网公司
3. 成本约束响应
	- 指标：工业电价、辅料成本、碳成本（如碳价）
	- 频率：月度/年度
	- 来源：各省电价文件、碳市场数据

你当前状态（已有基础）：
1. 电网工业电价（广东/云南/四川）
2. 电网平均碳排放因子（2010-2024相关材料、CEADs）
3. 电解铝产能数据（云南、四川、内蒙古、广西等）

### D. 清洁化结果（被解释变量）

核心问题：上游清洁化是否真实发生且可量化？

必收数据：
1. 单吨铝碳排放
	- 指标：企业层面吨铝碳排放、区域平均吨铝碳排放
	- 频率：年度
	- 来源：企业ESG、行业报告、LCA数据库
2. 总排放与减排量
	- 指标：产量 × 排放因子核算的地区总排放、相对基准减排量
	- 频率：年度/季度
	- 来源：你已有数据计算得到
3. 结构性结果
	- 指标：水电铝/绿电铝占比变化、高碳产能退出比例
	- 频率：年度
	- 来源：企业公告、行业数据库

你当前状态：
- 可做“地区核算版”结果（产量/产能 + 排放因子）。
- 待补“企业级吨铝碳排放”以增强证据强度。

### E. 控制变量与识别增强（计量模型必需）

核心问题：如何排除“不是消费驱动”的其他因素干扰？

必收数据：
1. 宏观与行业景气
	- 指标：GDP、固定资产投资、房地产与汽车景气、PPI
2. 政策冲击
	- 指标：能耗双控、限电、产能置换、环保督察时间点
3. 资源禀赋与水文波动
	- 指标：来水量、水电发电量、煤价波动

用途：
用于DID/事件研究/面板回归中的控制变量，识别“消费侧驱动”的净效应。

## 3. 结合你现有目录的“立即可执行”补数优先级

优先级 P0（先做，能马上形成实证闭环）
1. 把现有“产能 + 排放因子 + 电价”统一到省-年面板（至少2015-2025）
2. 增加“消费侧代理变量”
	- 对欧出口依赖度（铝相关）
	- 下游龙头绿色采购事件（企业层）
3. 构建被解释变量
	- 省级吨铝排放
	- 省级铝产业总排放

优先级 P1（第二阶段，增强机制识别）
1. 低碳铝溢价与订单数据
2. 企业级吨铝碳排放（重点公司：云铝、神火、魏桥等）
3. 绿电交易电量或绿证使用数据

优先级 P2（第三阶段，提升论文说服力）
1. 采购合同文本挖掘（碳约束条款）
2. 生命周期数据库（LCA）对比不同工艺路线
3. 海外客户与国内客户碳要求差异的案例数据

## 4. 推荐的数据表结构（建议新建）

1. demand_signal_panel.csv（消费端信号）
	- year, industry, export_eu_share, green_procurement_index, cbam_exposure
2. transmission_panel.csv（传导机制）
	- year, firm/province, low_carbon_al_purchase_share, premium, contract_constraint_dummy
3. upstream_response_panel.csv（上游响应）
	- year, province, capacity_running, output, grid_ef, green_power_share, industrial_power_price
4. outcome_panel.csv（清洁化结果）
	- year, province/firm, emission_intensity_tco2_per_tal, total_emission_tco2, clean_al_share

说明：四张表可通过 province + year（或 firm + year）关联，直接支持机制检验。
